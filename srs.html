3.6 Design Constraints

This section shall define constraints for the “Local Activities” application.

3.6.1 User-Input Constraints

This section shall define constraints for information inputted by a user.

3.6.1.1

The system shall require that user text input does not exceed 256 characters for descriptions, ratings and profile text fields.

3.6.1.2

The system shall require that user input does not exceed 128 characters for messaging text fields [interview].

3.6.1.3

The system shall not allow images or files to be attached in messaging text fields [interview].

3.6.1.4

The system shall not allow any description, profile, rating, or messaging text fields to be submitted while containing vulgar or inappropriate language.

3.6.2 Payment Constraints

This section shall define constraints for the way users will make payments.

3.6.2.1

The system shall require that the user use either PayPal, Apple Pay, Visa, or Mastercard for application payments.

3.6.3 Account Constraints

This section shall define the constraints of one account per person.

3.6.3.1

The system shall require users to create one account using Facebook, Google, or a phone number. []

3.6.4 Development Constraints

3.6.4.1

The system shall be written in the programming language Swift [].

3.6.4.2

The system shall be written using object-oriented development principles.

3.6.4.3

The system code shall be well documented with sufficient comments.

4.6.1 User-Input Constraints Verification

This section describes constraints on...

4.6.1.1

Verify that the system does not allow the user to input more than 256 characters for descriptions, ratings and profile text fields by testing each of these fields and attempting to input 257 or more characters.

4.6.1.2

Verify that the system does not allow the user to input more than 128 characters for messaging text fields by testing these fields and attempting to input 129 or more characters.

4.6.1.3

Verify that the system does not allow the user to insert images or files in messaging text fields by testing these fields and attempting to input images and files.

4.6.1.4

Verify that the system does not allow any description, profile, rating, or messaging text fields to be submitted while containing vulgar or inappropriate language by creating a word comparison list with the words to be excluded and testing this list in each of the fields by trying to input words from the list.

4.6.2 Payment Constraints Verification

4.6.2.1

Verify that the system requires the user to use either PayPal, Apple Pay, Visa, or Mastercard for application payments by testing the payment system and attempting to pay through different means.

4.6.3 Account Constraints Verification

4.6.3.1

Verify that the system requires the user to create one account using Facebook, Google, or a phone number by creating tests that attempt to create two or more accounts using the same Facebook, Google, or phone number.

4.6.4 Development Constraints Verification

4.6.4.1

Verify that the system is written in the programming language Swift by testing that the system doesn’t accept different languages such as C++, C#, Java, JavaScript, etc.

4.6.4.2

Verify that system is written using object-oriented development principles by checking that the code uses classes and objects instead of functional programing principles.

4.6.4.3

Verify that the system code is well documented with sufficient comments by checking the numbers of comments averaging about 1 comment around every 50 lines of code.
