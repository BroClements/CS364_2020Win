Top Down Technique Report

Overview of the Technique

Technique: Top Down

The Top-Down Approach is a technique that produces specific requirements by breaking down a system to understand the details of a requirement. To begin, you start with some general or high-level features or requirements and then systematically break them down into increasingly smaller and more specific requirements. This assures that the resultant requirements are all focused on the main goals and are specific enough to be easily implemented.

Pros Regarding Top Down:

  • It helps align the project goals with customer expectations.
  • Provided a hierarchical requirement flows covering the details of the requirements.

Cons Regarding Top Down:

  • Communication can easily become unclear. Vague requirements and expectations can rapidly lead to confusions which can eventually result in project failure.
  • Obscure requirements that do not directly fall in the scope of one of the major requirements may be overlooked and left out of the project scope.

Application to the Project:

This technique will provide a list of requirements in an ordinary matter from most general to most specific. The team has met together and broken down some of the requirements identified through other techniques as well some of the information provided on the application description. The implementation of this technique will benefit the project by finding the details of specific requirements of the application.

Carrying out the technique

  1. Identify a minimum of three high-level requirements.
  2. Analyze requirements, breaking them down into more basic components.
  3. Continue breaking down components until the most basic attributes are identified and collect the data from them.
  4. Record into a word document and report findings.

Conclusion

The top down technique has provided the project with a set of unique requirements. By focusing first on the overall objective of the application and progressively breaking things down into smaller and more specific chunks, ultimately resulting in a set of basic requirements for the project. These requirements are critical to the success of the overall project. The top down technique is effective in finding specific requirements that can be more easily implemented and ensures that the requirements all work together to accomplish the overall goal of the project.

Appendix: Raw Data Collected

  • User- Experience:
    • Aesthetics:
      • Friendly/Psychological Colors
        • Colors change depending on type of activity (dates/with friends)
      • Font
        • The fonts are easy read and appealing to the user
            Sans-Serif fonts should be used for readability
    • Tutorial:
      • Screenshots Demonstration
        • Screenshots demonstrating how to use the app
      • Tooltips
        • Tooltips that described the functionality of the application’s features
      • Intro video
        • Video Demonstration of the app features
    • Intuitive:
      • Simplicity
        • The degree to which the application is plain or easy to understand.
        • Limited number of buttons
      • Follows Established Norms
        • The degree to which the application follow popular tends.
    • Navigation
      • Auction screen for business to bid on priority listing spots
      • Saved deals feature
        • Users should have the opportunity to save deals they are interested onto a favorites tab
      • Suggested feed
        • The application should keep track of the kinds of activities a user is interested in and then offer similar activities to them in their feed.
      • Tags:
        • Events can be assigned tags. Events can be filtered by tag.
      • Search bar:
        • The application should have a search bar when the user is selecting an activity to create so that they can search generally.
  • Functionality
    • Fast performance
      • The application should utilize background threads to retrieve and store information in order to not interfere with the MAIN UI Thread.
    • Effective Data Storage
      • An appropriate database format is selected for storing information
    • Reliability
      • App does not crash; error checking is consistent throughout the code to ensure robustness.
      • Consistent updates to maintain compatibility with current phone types.
    • Security
      • User Accounts
        • Information is only available to those who should have access to it
    • Responsiveness
      • Async server requests:
        • All requests to the server should he made asynchronously, freeing the user from the server.
      • Server size/abilities
        • The server should be large enough to handle all requests in a timely manner.
  • Create Events
    • Businesses can list events that they offer
      • Businesses compete (with price) to give priority to their events.
    • Users can post events that they are planning (this can consume business listings)
      • Users can control who can join their event.
