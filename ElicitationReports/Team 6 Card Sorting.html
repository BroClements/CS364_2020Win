Card Sorting

Overview of the Technique

This report will detail the purpose and goals of card sorting as a requirements elicitation technique and the plan for implementing this technique. It will also summarize patterns found in the responses of volunteer participants in the research which will give an idea of how the users would expect the app’s design to be like. The data acquired and analyzed in this report will help in the architecture, design and organization of the application, providing a usable, learnable and logical system.

Card Sorting Plan

Purpose

The purpose of this technique is to create categories of features in the application. According to useability.gov, “In a card sorting session, participants organize topics into categories that make sense to them and they may also help you label these groups [1].” In our application we will use this to determine what categories and features will be grouped together and from there we will organize the categories into separate pages and locations on the application.

There are two primary ways to use the Card Sorting Technique, opened and closed. An article from newsignature.com details open card sorting by saying, “In an open card sorting session, participants create their own names for the categories. This helps reveal not only how they mentally classify the cards, but also what terms they use for the categories [2].” It further explains that closed card sorting is, “participants are provided with a predetermined set of category names. They then assign the index cards to these fixed categories. This helps reveal the degree to which the participants agree on which cards belong under each category [2].” We will be choosing one method to follow in this case.

Technique

We plan to use the closed card sorting technique and provide a list of categories and features to be sorted by the users and one other team from the project. We will compile a list of features and create a list of categories for these features and then present them to the participants for sorting. We will collect our features from our own research and the other team’s research. The categories will be as follows:

  • Events
  • People
  • Schedules
  • Messaging

Steps

  1. We will first collect a list of features from the other teams in the project
  2. Next, we will create a list of categories to sort the features into
  3. Next, we will use the optimal workshop website to create the categories and features (https://www.usabilitest.com/
  4. We will then send a link to the participants to organize the features
  5. We then will collect the data and store it in an excel spread sheet

Collected Data

Data will be collected from the optimal workshop website and then downloaded to an excel spreadsheet for future reference and follow-up. The data being collected will be from the users of the application. The users will sort features into different categories and organize it in ways that seem logical to them.

Analyzing the Data

We will then compare our answers and create a list of features and categories that were most used by the users. This will be our final result for use in the application.

Conclusions

"Card sorting is a method used to help design or evaluate the information architecture of a site[1]." This allows us to create categories and pages within the application so that the user can be easily navigated. We chose to implement a closed card sort so that we can predefine a set of categories. This technique will help us to learn how “users sort content items into each category [1]." To implement this technique, our team used usabilitest.com to create categories and features within the application. This website also helped us to collect and store the data for our card sort. Once we collected sufficient data, we will analyze the data by comparing features and categories to find out the more popular options.
